# Development Guide

For developers and AI agents working on the Caros code. Pairs with
[ARCHITECTURE.md](./ARCHITECTURE.md). Also read the root [CLAUDE.md](../CLAUDE.md) for the
short hard-rules version.

## Migrations — the workflow that actually applies here

- **Migrations are reviewed, then applied by hand** in the Supabase **SQL editor**. There is
  **no** `supabase db push` / Docker workflow in this project (every migration header says so).
- Every file in `supabase/migrations/` is written **idempotent** (`IF [NOT] EXISTS`,
  drop-before-create on policies, `DO`-block guards on constraints/FKs) and documents its
  **apply order** in a header comment. Canonical order (from the headers):
  ```
  baseline_schema → extend_market_enums → harden_my_tenant_id → rls_policies
  → p4_public_read_policies → p5a_cars_status → p5a_rental_terms
  → p5a_storage_car_images → p5b_leads_bookings → p6_tenant_sections
  → p2_5_lead_types → p2_5_4b_tenant_pages → p2_5_4b2_tenant_content
  → qa2b_lead_whatsapp_flag → e1e3_car_brands → e4_car_content_per_locale
  → p7_car_financing → p8_installment_monthly
  ```
- **`ALTER TYPE … ADD VALUE` cannot run in a transaction that also *uses* the value.** Keep
  enum-extension migrations add-only (see `20260614090000_extend_market_enums.sql`).
- After a schema change, if PostgREST serves a stale cache (a new column/enum reads as "does
  not exist" via the API), run:
  ```sql
  NOTIFY pgrst, 'reload schema';
  ```
  (Several migrations already include this.)
- **SECURITY DEFINER functions must pin `search_path`** (`set search_path = ''` + fully
  schema-qualified names). See `20260614092000_harden_my_tenant_id.sql` and
  `mark_latest_lead_whatsapp`.

### Regenerating Supabase types (Windows — UTF-8 only)

`lib/supabase/database.types.ts` is generated by the Supabase CLI. On Windows PowerShell,
**force UTF-8** — the default `>` redirect writes **UTF-16 LE (BOM)**, which breaks the TS
build with confusing parse errors.

```powershell
# CORRECT — UTF-8
supabase gen types typescript --project-id <ref> --schema public |
  Out-File -Encoding utf8 lib/supabase/database.types.ts

# WRONG — default redirect writes UTF-16 → build breaks
supabase gen types typescript ... > lib/supabase/database.types.ts
```

`lib/supabase/mappers.ts` validates DB enum values against the generated `Constants`, so after
a regen the mappers stay the single bridge between `database.types.ts` and `types/vehicles.ts`.

## Key conventions (do these)

- **Leads are write-only for the public.** When inserting a lead as anon, **never** chain
  `.select()` — anon has no SELECT policy on `leads`, so `INSERT … RETURNING` is denied
  (`42501`) and the row rolls back. See `lib/leads/submit.ts`. Resolve `tenant_id` from the
  `x-tenant-id` header server-side, **never** from client input.
- **UUIDs: use `safeRandomUUID()`** from `lib/utils/uuid.ts` — never call
  `crypto.randomUUID()` directly. `crypto.randomUUID` only exists in **secure contexts**
  (HTTPS/localhost); on a plain-HTTP origin like `dealer2.lvh.me` it is `undefined` and throws,
  crashing the page. The util falls back to `crypto.getRandomValues` (available on HTTP). Used
  by the car form + `lib/storage/carImages.ts`.
- **Per-tenant data flows through providers / server fetchers, not static config.** Read
  tenant data via `useTenantFeatures()` and the other `Tenant*Provider` hooks (client) or
  `getTenantConfig()` / `getStorefrontFeatures()` (server). The storefront is fully
  Supabase-backed via `lib/supabase/queries.server.ts` — **do not reintroduce a static
  `data/cars.ts`** (it was removed; the code and `page.tsx` comments call this out).
- **Pick the right Supabase client** (see [ARCHITECTURE.md](./ARCHITECTURE.md) §4):
  `createPublicServerClient()` for storefront anon reads; `createSupabaseServerClient()`
  (`lib/supabase/server.ts`) for dashboard reads/writes as the logged-in user;
  `createServerClient()` (service role, bypasses RLS) only for scripts / trusted mutations;
  `createBrowserClient()` for client-side auth.
- **Server Actions revalidate the storefront.** Inventory mutations call `revalidatePath('/',
  'layout')` (plus the dashboard paths) so add/edit/delete/status changes appear on the public
  site without a hard refresh (see `revalidate()` in `dashboard/cars/actions.ts`).
- **Car images** are compressed to WebP in the **browser** then uploaded to the `car-images`
  bucket at path `{tenant_id}/cars/{car_id}/{uuid}.webp` (`lib/storage/carImages.ts`) — the
  path prefix is what storage RLS checks.

## Gotchas / hard-won lessons (from the actual code)

| Symptom | Cause | Fix |
|---|---|---|
| TS build fails with garbled/parse errors in `database.types.ts` | type-gen written as **UTF-16** by `>` redirect | regenerate with `Out-File -Encoding utf8` |
| New column/enum "does not exist" via the API after a migration | PostgREST **schema cache** stale | `NOTIFY pgrst, 'reload schema';` |
| Page crashes: `crypto.randomUUID is not a function` on a `.lvh.me` (HTTP) origin | `randomUUID` missing on a non-secure context | use `safeRandomUUID()` |
| Public lead insert silently fails / rolls back (`42501`) | `.select()`/RETURNING on an anon `leads` insert | write-only insert, no `.select()` |
| Sign-in appears to "hang" (dashboard bounces to login) | session in `localStorage`, invisible to the server | use the `@supabase/ssr` cookie client (`createBrowserClient` / `createSupabaseServerClient`) |
| Storefront 404 with a real subdomain | subdomain doesn't map to an **active** tenant (no silent default fallback) | check `tenants.slug` / `subdomain` / `active`; in prod check `NEXT_PUBLIC_ROOT_DOMAIN` |
| Subdomain resolves to the wrong / default tenant in prod | `NEXT_PUBLIC_ROOT_DOMAIN` unset → host treated as apex/`*.vercel.app` | set `NEXT_PUBLIC_ROOT_DOMAIN` and redeploy |
| Server component renders Arabic when URL is `/en` | next-intl `getLocale()` can't resolve without its own middleware | pages call `setRequestLocale(locale)` from the route param (already done in layouts/pages) |
| A "body type" shows `electric` | legacy `car_category='electric'` overlaps fuel | app code flags/excludes it; full fix is the P3 remap |
| Financing monthly shows the **rental** price / a car appears to have "two monthly" values | `installment_monthly` (financing) conflated with `price_monthly` (rental) | keep them separate: `pricing.installmentMonthly` ← `installment_monthly`, `pricing.monthly` ← `price_monthly` (P8). `down_payment` + `installment_monthly` are the financing pair |

## Before you merge — checklist
- [ ] `npm run build` green
- [ ] `npm run lint` clean
- [ ] `npx tsx scripts/test-rls.ts` → all passed (set `TEST_TENANT_A_*` for the full
      cross-tenant proof; an incomplete run reports as failed by design)
- [ ] migrations idempotent + header-documented; applied to the DB; `NOTIFY pgrst` if needed
- [ ] types regenerated (UTF-8) if the schema changed
- [ ] tested light/dark and a relevant tenant type (sale-only / rental-only / hybrid)
